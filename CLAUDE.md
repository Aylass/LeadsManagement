## Development Commands

### Frontend (Next.js 15 + React 19)
```bash
cd frontend
npm run dev      # Start dev server on http://localhost:3000
npm run build    # Production build
npm run lint     # ESLint
```

### Backend (Express + TypeScript)
```bash
cd backend
npm run dev      # ts-node-dev with hot reload on port 10000
npm run build    # Compile TypeScript to dist/
npm run check    # Type-check without emitting (tsc --noEmit)
npm start        # Run compiled dist/start.js
```

### Environment Variables
- **Frontend** (`.env.local`): `NEXT_PUBLIC_API_URL=http://localhost:5000/api`
- **Backend** (`.env`): `MONGODB_URI`, `PORT` (hardcoded to 10000 in start.ts), `NODE_ENV`, `FRONTEND_URL`

> Note: The backend port is hardcoded to `10000` in `start.ts`, not read from `PORT` env var.

## Architecture

This is a monorepo with two independent packages:

```
Lead-Management/
├── frontend/   # Next.js App Router app
└── backend/    # Express REST API
```

### Backend
- **Entry**: `backend/start.ts` — sets up Express with CORS (whitelist via `FRONTEND_URL`), Helmet, rate limiting (100 req/15min), and mounts all routes
- **Data layer**: Mongoose + MongoDB Atlas. Single model: `backend/models/Lead.ts`
- **Lead schema**: `name`, `email` (unique, lowercase), `status` (enum: New | Engaged | Proposal Sent | Closed-Won | Closed-Lost). Has text index on name+email for search.
- **Routes** (`backend/routes/leadRoutes.ts`): full CRUD on `/api/leads` + `/api/leads/stats`
- **Controller** (`backend/controllers/leadController.ts`): all query logic lives here — pagination (max 50/page), regex search, status filter, sort
- **Types** (`backend/types/index.ts`): shared backend types for `LeadQuery`, `LeadResponse`, `ApiResponse`

### Frontend
- **Framework**: Next.js 15 App Router, React 19, TypeScript
- **UI**: Tailwind CSS + shadcn/ui (Radix UI primitives). Components in `frontend/components/ui/` are auto-generated by shadcn CLI — do not manually edit them.
- **API client** (`frontend/lib/api.ts`): singleton `apiClient` class wrapping `fetch`. All API calls go through it. `NEXT_PUBLIC_API_URL` must be set.
- **Lead types** (`frontend/types/lead.ts` and inline in `frontend/lib/api.ts`): `LeadStatus` union type is defined in `lib/api.ts`.
- **Pages**:
  - `/` — Dashboard with stats and recent activity (`frontend/app/page.tsx`)
  - `/leads` — Paginated leads list with search/filter/sort; state synced to URL search params (`frontend/app/leads/leads-client.tsx`)
  - `/leads/new` — Create lead form (`frontend/app/leads/new/page.tsx`)
- **State management**: URL search params are the source of truth for leads list filters (page, limit, search, status, sortBy, sortOrder). Search input is debounced 500ms before updating URL.

### API Contract
All responses follow `{ success: boolean, data?: T, error?: string, message?: string }`.

Leads list endpoint: `GET /api/leads?page=1&limit=10&search=&status=all&sortBy=createdAt&sortOrder=desc`

The `/api/leads/stats` endpoint returns `{ total, statusCounts, conversionRate }` computed via MongoDB aggregation.
